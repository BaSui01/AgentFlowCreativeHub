# AgentFlowCreativeHub 项目规范

## 🌐 基本原则

### 语言交互规范

- **始终使用中文交互**
- AI 助手与用户的所有对话、文档、代码注释均使用中文
- 确保沟通清晰、易于理解

## 📋 文档管理规范

### 文档存放位置

- **所有项目文档统一存放在 `docs/` 文件夹**
- 按照功能模块或文档类型进行子目录划分

### 文档命名规范

- **使用中文命名**
- 文件名清晰表达文档内容
- 示例：`用户指南.md`、`API接口文档.md`、`架构设计.md`

### 文档创建原则

- ⚠️ **禁止 AI 自动创建文档**
- ✅ **仅在用户明确要求时创建文档**
- 避免生成不必要的总结性 `.md` 文件
- 重要笔记使用 `notebook-add` 工具记录

### 文档状态标记

- **所有文档任务使用状态标记**
  - `[ ]` - 未完成
  - `[x]` - 已完成
- 便于追踪文档完成进度

### 文档更新方式

- **增量更新** - 不覆盖已有内容，追加新内容
- **渐进完善** - 逐步补充完善文档细节
- **保持同步** - 代码变更时及时更新相关文档

## 🎯 使用场景

### 何时创建文档

- ✅ 用户明确要求生成 README
- ✅ 用户要求编写 API 文档
- ✅ 用户要求创建使用指南
- ✅ 用户要求输出技术规范
- ✅ 用户要求编写设计文档

### 何时使用 Notebook

- 📝 记录代码易碎点和注意事项
- 📝 记录已修复的 bug 防止复现
- 📝 记录非显而易见的依赖关系
- 📝 记录临时解决方案和约束条件
