## 目标
在现有权限基础上继续完善前端管理体验，覆盖剩余的角色同步、权限枚举展示一致性以及管理端 UX 细节。

## 建议步骤
1. **角色/用户数据同步完善**
   - 向后端确认/补充 `GET /api/tenants/:id/users/:userId/roles` 或在列表接口返回角色数组，前端 `RoleManagementPage` 在初次加载时拉取全部用户角色映射。
   - 用户角色列显示真实数据，并在刷新或 mutate 成功后统一走 React Query 缓存，而非本地 state。

2. **权限字典与国际化**
   - 将 `tenant/internal/permissions_seed.go` 中的权限列表导出为 API／静态 JSON，前端统一引用（避免硬编码 category 名称）。
   - 在 UI 中显示 `name`/`description`，并在无本地化时回退英文。

3. **操作审计与提示**
   - 为角色创建、删除、分配等操作加上 Result notification + undo（若后端支持），减少误操作。
   - 管理端页面顶部展示当前用户权限摘要，明确操作范围。

4. **管理端 UX 优化**
   - 在 `/admin/operations` 增加命令历史、审核统计等卡片，通过 React Query 查询 `/api/commands` 或审核统计接口。
   - 为命令执行/审核面板提供搜索、筛选与分页能力，提升大量数据时的可用性。

完成上述改进后再进入实现阶段。
