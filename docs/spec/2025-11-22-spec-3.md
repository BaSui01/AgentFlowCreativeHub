## 目标
基于已实现的权限管理与运营控制台，进一步提升跨端一致性、数据准确性以及可观察性，确保管理员能够完整掌握成员授权与命令执行状况。

## 计划步骤
1. **同步后端角色接口到 UI**
   - 在角色管理页的成员标签中使用 `tenantUserRoles` Query 数据，支持刷新按钮触发单独的 `getUserRoles` 请求。
   - 对接新 GET 接口时，对于无数据的用户显示“无角色”，避免误判。

2. **权限字典统一与国际化**
   - 将后端 `permissions_seed.go` 中的分类/名称导出为 JSON 或常量映射供前端消费，确保类别标签不会硬编码。
   - 在 UI 中显示 `description`，并补充英文 fallback。

3. **操作反馈与审计提示**
   - 角色 CRUD 与分配操作除了 toast，还记录最近一次更新摘要，便于管理员回溯。
   - 运营控制台顶部展示当前用户角色、权限数量、上次登录时间（依赖后端用户信息）。

4. **命令与审核可观察性**
   - 运营控制台新增：命令列表分页、筛选（状态/Agent），以及审核历史面板。
   - 使用 React Query `useInfiniteQuery` 拉取 `/api/commands`，在命令卡片中提供重试/终止入口（若后端支持）。

5. **技术债梳理与测试**
   - 为新的角色映射 Hook 与运营统计组件补充单元测试。
   - 联合运行 `npm run lint && npm run test` 与 `go test ./...`，保持前后端一致性。

确认方案后再进入实现阶段。
