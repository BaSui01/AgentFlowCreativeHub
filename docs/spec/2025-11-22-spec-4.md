## 目标
完成剩余的“统一权限字典”高优先级任务，并进一步加强运营控制台的观测与回溯能力，确保 RBAC 配置与运维可视化形成闭环。

## 计划步骤
1. **后端权限字典统一导出**
   - 在 `internal/tenant/permissions_seed.go` 中生成结构化权限清单（含 code、名称、分类、描述、多语言文案）。
   - 新增 `/api/tenant/permissions/catalog` 接口返回完整字典，支持版本号/缓存头，前端仅依赖该接口。

2. **前端权限引用改造**
   - `use-authorization` 引入字典来源，确保分类标签、描述等均来自后端返回数据（含中文/英文 fallback）。
   - RoleManagementPage 的 `PermissionChecklist` 改为按 `categoryOrder` 渲染，并增加搜索过滤。

3. **命令与审核可观察性深化**
   - 运营控制台命令列表：追加“运行耗时、Token 消耗、队列位置”列，并提供导出 CSV 功能。
   - 审核历史面板扩展显示审核人、操作类型，支持按时间区间筛选。

4. **审计与操作记录**
   - 后端记录角色 CRUD 与命令执行的审计日志（若已有中间件则补充字段），前端在管理页展示最近 10 条操作。

5. **测试与验证**
   - 新权限字典接口及 Hook 编写单元测试，校验多语言 fallback。
   - 更新前端、后端测试快照并运行 `npm run lint && npm run test`、`go test ./...`。

确认该计划后进入实施阶段。
